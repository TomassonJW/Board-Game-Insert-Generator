# Logs d'avancement

Ce dossier contient les journaux d'avancement significatifs du projet.

Un log est utile quand une mission :

- change l'orientation du projet ;
- termine une phase ou un jalon ;
- modifie le statut reel ;
- ajoute une decision de pilotage ;
- constate un risque important ;
- documente un resultat de validation ou d'impression reelle.

## Format recommande

```markdown
# YYYY-MM-DD - Titre court

## Contexte

Pourquoi le log existe.

## Changements

- Changement 1.
- Changement 2.

## Verifications

- Commande lancee : resultat.

## Impact

Ce que cela change pour le prochain agent.

## Suivi

- Carte backlog suivante.
```

## Regles

- Ne pas utiliser les logs pour remplacer `docs/STATUS.md`.
- Ne pas y stocker de secrets.
- Garder les logs factuels et datables.
- Ajouter un log seulement pour les missions significatives, pas pour chaque
  micro-correction.
